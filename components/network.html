Network介绍

网络是显示网络和由节点和边组成的网络的可视化。可视化易于使用，并支持自定义形状、样式、颜色、大小、图像等。网络可视化可以在任何现代浏览器上流畅地工作， 最多支持几千个节点和边。为了处理更多的节点，Network提供了集群支持。网络使用HTML画布进行渲染。
从4.0开始，网络由处理网络特定部分的各个模块组成。这些模块有自己的文档、选项、方法和事件，您可以通过单击下面列表中的模块来访问它们。

Network中包含的模块有哪些

autoResize:true

【boolean】自动调整：默认为true。 如果为true，网络将自动检测其容器的大小调整，并相应地重绘自身。如果为false， 则可以在使用函数redraw（）和setSize（）调整其容器大小后强制网络重新绘制

 height:'100%'

【String】画布的高度。 immediate可以是百分比或像素，一般在dom元素上设置

 width:'100%'

【String】画布的宽度。 可以是百分比或像素，一般在dom元素上设置

 locale:'en'

【String】语言设置。默认为en。

 locales:{}

clickToUse:false
【boolean】默认是false；它将仅在激活时对鼠标和触摸事件作出反应。激活时，network（dom元素盒子）周围会显示蓝色阴影边框。 通过单击将network设置为活动状态，然后通过单击网络外部或按ESC键再次将其更改为非活动状态。通过设置该属性，可以清晰的看到当前的network 是否是被激活状态，从而可以明晰事件是否可以执行（尤其是针对鼠标hover的事件，尤其明显）
configure:{}
【Object】默认是
configure ：{
enabled ：true ，(打开或关闭配置界面。这是一个可选参数。如果未定义但定义了此对象的任何其他属性，则将其设置为true。)
filter ：'nodes，edges' ，(可接受的值类型：字符串，数组，布尔值，函数)
container ：undefined ，(允许将配置列表放在另一个HTML容器中，而不是放在network下面),这里接受dom元素，且为原生js获取的dom元素 使用jquery获取的元素不被支持，原理是，这里的插入使用的原生的appendchild方法；例如document.getElementById("#config");
showButton ：true(显示配置程序底部的生成选项按钮。通过此按钮，可以看到option中配置的所有参数)
}
filter的相关配置详情
filter：true/false 当布尔值为true时，为您提供所有选项，false将不显示任何选项
filter：['nodes','edges'] 当值为数组时，配置数组中的组合，可传值有【nodes, edges, layout, interaction, manipulation, physics, selection, renderer】
filter：'nodes，edges' 当值为字符串时，配置字符串中的组合，可传值 同数组
filter：function （option ,path ）{ path.indexOf('physics' )！== - 1 ; } 提供函数时，您会收到两个参数。选项对象中的选项和选项路径。 如果返回true，则选项将显示在配置程序中
